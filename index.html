01.Sass 介紹

Sass 是 css 的預處理器，目的在減少 css 的重工。

Sass 有變數、函式、流程控制、嵌套、繼承、mixins 這些功能讓我們使用，所以比 css 在寫樣式時更有彈性。

Sass 可以更有效率維護樣式的結構。

Sass 需要編譯成原生的 css 才能使用，瀏覽器看不懂 Sass。

03. 變數

宣告使用 $，變數位置即是作用域，就像 JS 的 let 宣告，寫在大括號裡面是區域變數，最外面是全域變數。

變數底線 _ 和連結線 - 兩者是一樣的變數，例如 $bg_dark、$bg-dark。

使用變數可以維持網站的一致性，節省修改成本，只需要修改變數，不需要在 css 修改。

04. 流程控制

@if、@else if、@else

判斷條件可以小括號，也可以不括號

使用 and、or、not 處理判斷運算式的結果是 true 或 false

歐拉

@for 迴圈有兩種 through 與 to

@for $i from (start number) through (end number){...}，包含 end number

@for $i from (start number) to (end number){...}，不包含 end number

05. 巢狀(嵌套,nested)

& 佔位符號可以替換父選擇器。

勿過度使用嵌套，考慮用最少的選擇器，選擇到想要的位置，盡量在 3~4 層。

過度使用嵌套，編譯後選擇器會有過長的問題，不易閱讀及維護，造成 CSS 檔案容量較大。

06. 繼承

繼承可以使用 css 選擇器或 % 佔位選擇器來命名

使用 css 選擇器當作繼承，會被編譯到 css，% 佔位符屬於 scss 的範圍，不會被編譯到 css。

有共同樣式可使用繼承，會被聚集成群組選擇器，樣式碼會較少。
